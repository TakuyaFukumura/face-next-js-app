# 仕様書

## 1. 概要

本リポジトリは **Next.js 16 / React 19 / TypeScript** を用いた Web アプリケーションです。  
現在の主機能は、ブラウザ上でカメラ映像を取得し、`face-api.js` を使って **顔検出・表情推定・年齢推定・性別推定** をリアルタイムで行うクライアントサイドアプリです。

推論に必要なモデルは `public/models/face-api/` から読み込まれ、映像や推論結果はサーバーへ送信されません。  
加えて、初期実装由来の **SQLite + `/api/message`** のサンプル API/DB 機能もリポジトリ内に残っています。

## 3. 主な機能

### 3.1 リアルタイム顔検出・属性推定

- ブラウザの `getUserMedia` で前面カメラ映像を取得
- `TinyFaceDetector` による顔検出
- 表情推定
- 年齢推定
- 性別推定
- 検出人数の表示
- 顔領域のバウンディングボックスと推定ラベルの描画

### 3.2 状態表示

アプリは以下の状態を持ち、`StatusPanel` で表示します。

- `idle`
- `loadingModels`
- `modelError`
- `startingCamera`
- `cameraError`
- `permissionDenied`
- `analyzing`
- `noFace`

### 3.3 ダークモード

- ライト/ダークの手動切替
- 選択状態を `localStorage` に保存
- `html` 要素へ `dark` クラスを付与してテーマ反映

### 3.4 プライバシー表示

画面下部に以下の注意事項を表示します。

- 映像・推論結果はサーバー送信しない
- 推定値には誤差がある
- 個人識別は行わない
- センシティブ用途を禁止
- HTTPS 環境が必要（`localhost` を除く）

### 3.5 付随 API / DB 機能

- `GET /api/message` で SQLite から最新メッセージを返却
- DB ファイルが存在しない場合は `data/app.db` を自動生成
- `messages` テーブルを自動作成
- 初期データとして `Hello, world.` を投入

## 5. 顔認識処理の仕様

### 5.1 モデルロード

`useFaceApiModels` がマウント時に以下 3 モデルを非同期ロードします。

- `tinyFaceDetector`
- `faceExpressionNet`
- `ageGenderNet`

読み込み元:

- `/models/face-api/tiny_face_detector_model-*`
- `/models/face-api/face_expression_model-*`
- `/models/face-api/age_gender_model-*`

### 5.2 カメラ制御

`useCamera` の責務:

- `videoRef` の提供
- カメラ起動/停止
- `playing` イベント検知によるストリーミング状態管理
- エラー分類

エラー種別:

- `permissionDenied`
- `notFound`
- `unsupported`
- `unknown`

### 5.3 推論ループ

`useFaceDetection` が以下条件で動作します。

- モデル読込済み
- カメラ映像再生中

主な仕様:

- 推論間隔: `300ms`
- 検出閾値: `0.5`
- `detectAllFaces(...).withFaceExpressions().withAgeAndGender()` を実行
- 検出結果を UI 用に正規化
- canvas に顔枠とラベルを描画
- 停止時は検出結果とオーバーレイをクリア

### 5.4 描画内容

`drawDetections` では各顔に対して以下を表示します。

- 顔枠
- 表情名とスコア
- 推定年齢
- 推定性別と確率

ラベル位置は顔枠の上部を優先し、領域不足時は下部へ描画します。

## 6. データ仕様

### 6.1 正規化後の検出結果

`NormalizedDetection` は以下の情報を保持します。

| 項目 | 説明 |
| --- | --- |
| `box` | 顔領域の座標とサイズ |
| `expression` | 最も確率の高い表情ラベル |
| `expressionScore` | 表情スコア |
| `age` | 推定年齢 |
| `gender` | `male` / `female` |
| `genderProbability` | 性別推定の確率 |

### 6.2 SQLite テーブル

`messages` テーブル:

| カラム | 型/内容 |
| --- | --- |
| `id` | INTEGER PRIMARY KEY AUTOINCREMENT |
| `content` | TEXT NOT NULL |
| `created_at` | DATETIME DEFAULT CURRENT_TIMESTAMP |

## 7. API 仕様

### 7.1 GET `/api/message`

**概要**  
SQLite から最新メッセージを取得して返却します。

**正常時レスポンス例**

```json
{
  "message": "Hello, world."
}
```

**異常時レスポンス例**

```json
{
  "error": "メッセージの取得に失敗しました"
}
```

## 9. テスト仕様

## 10. CI/CD

GitHub Actions (`.github/workflows/ci.yml`) により、push 時に以下を実行します。

1. `npm ci`
2. `npm run lint`
3. `npm run build`
4. `npm test`

対象 Node.js は `20.x` です。

## 11. 実行・運用上の前提

- Node.js 20 系を想定
- カメラ機能は対応ブラウザが必要
- 本番環境では HTTPS が必要
- 顔認識モデルファイルが `public/models/face-api/` に配置されている必要がある

## 12. 補足

README には SQLite のサンプルアプリ説明が多く残っていますが、実装上の主アプリケーションは **リアルタイム顔検出・属性推定 Web アプリ** です。  
一方で、SQLite と `/api/message` は補助的な既存機能として引き続き同居しています。
